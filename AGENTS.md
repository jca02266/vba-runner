# Repository guidance

## Commit messages

Use a type-prefixed, concise one-line summary, followed by a blank line and a
body beginning on the third line. Select a type that makes the change category
clear, such as `Fix` for bug fixes, `Add` for new functionality, `Improve` for
behavioral improvements, `Docs`, `Test`, `Chore`, or `Verify`. Make the summary
name the affected behavior and outcome; use the body to explain the reason and
the material implementation details. Avoid vague messages that require reading
the diff to understand the change.
