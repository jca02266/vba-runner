# BNF Parser Gap Audit スキル

BNF.md（MS-VBAL 仕様書から抽出済み）とパーサー実装を照合し、未対応の構文をリストアップする。
`/bnf-audit` を実行するたびに次の未分析バッチを自動選択して調査・記録する。

## 手順

### Step 1: 状態ファイルを読んで次のバッチを決定

`docs/internals/PARSER_BNF_AUDIT_STATE.md` を読む。
「次に実行するバッチ」欄を確認し、対応する行番号範囲を把握する。
すべてのバッチが「✅ 完了」なら「全バッチ完了」と報告して終了。

### Step 2: BNF と対応パーサーコードを読む

`docs/internals/BNF.md` の該当行範囲を Read ツールで読む。

`src/engine/parser.ts` から、対象 BNF セクションに対応する parse* 関数を
grep で特定してから Read で読む。
（例: `grep -n "private parse" parser.ts | grep -i "option\|declare\|type\|enum"`）

### Step 3: サブエージェントに並列で gap 分析させる

バッチ内のセクション数に応じて 2〜3 エージェントに分割して並列起動する。
各エージェントへのプロンプト要件:
- 対象 BNF ルールの全文をプロンプトに埋め込む（ファイルパスだけでなく内容ごと渡す）
- 対応するパーサー実装コードを同様に埋め込む
- 各節について「✅ OK / ⚠️ 部分対応（説明）/ ❌ 未対応（説明）」を1行で返させる
- 問題のある節には「原因となる VBA コード例を1行」添えさせる
- ツール使用禁止の指示を入れる（コードはプロンプトに埋め込み済みのため）

### Step 4: 結果を PARSER_BNF_GAPS.md に追記

`docs/internals/PARSER_BNF_GAPS.md` に今回のバッチ結果を追記する。
フォーマットは既存バッチ（§5.4.x）と同じ見出し・表形式を踏襲する。

### Step 5: 状態ファイルを更新

`docs/internals/PARSER_BNF_AUDIT_STATE.md` を更新する:
- 今回処理したバッチを「⬜ 未」→「✅ 完了」に変更
- 「次に実行するバッチ」欄を次の未完了バッチに更新

### Step 6: ユーザーに報告

以下を簡潔に報告する:
- 今回のバッチで発見した問題（🔴高 / 🟠中 / 🟡低）のサマリー
- 残りバッチ数と次回対象
- 所要時間・トークン量（task-notification の値から）

## 注意事項

- サブエージェントはコードを読みに行く傾向があるため、プロンプトに必要なコードを埋め込んでも
  ツール使用が増える。許容範囲だが、プロンプトを自己完結させることで精度を上げられる
- §3.x（字句解析レベル）はレキサー（lexer.ts）の問題。パーサーとは別に扱う
- 発見した gap を修正するかどうかはユーザーが判断する（スキルは調査のみ）
