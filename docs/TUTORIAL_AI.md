# TUTORIAL_AI.md — AI への指示だけで完結するチュートリアル

## 対象読者

コマンドや実装の詳細を知らなくても、AI コーディングエージェント（Claude Code 等）への**指示文だけ**で、Excel VBA の抽出・確認・リファクタリング・機能追加・書き戻しを完結させたい人向けです。

このドキュメントには `bash` コマンドや TypeScript コードは登場しません。各ステップで「あなたが AI に伝える指示文の例」だけを示します。コマンドの実行・テストコードの作成・モックの実装といった作業は、すべて AI が `vba-runner` 一式（実行エンジン・解析ツール・抽出ツール）とリポジトリ内のガイド文書（`FOR_AI.md` / `MOCK_GUIDE.md` 等）を使って行います。

> コマンドの詳細を自分の手で確認したい場合は [TUTORIAL.md](TUTORIAL.md) を参照してください。

## 前提

- 本リポジトリ（vba-runner）を使える AI コーディングエージェント（Claude Code 等）が動いていること
- リファクタリング・機能追加したい `.xlsm` ファイルがあること

---

## 0. インストール

### Node.js のインストール

[Node.js 公式サイト](https://nodejs.org/) から LTS 版のインストーラーをダウンロードして実行してください。

Windows 10 / 11 では winget を使ってインストールすることもできます（PowerShell で実行）:

```powershell
winget install OpenJS.NodeJS.LTS
```

インストール後、PowerShell を再起動してバージョンを確認します:

```powershell
node --version
npm --version
```

### ツールのインストール

PowerShell で以下を実行します:

```powershell
# VBA ソースの抽出・書き戻しツール
npm install -g vba-extractor

# VBA 実行エンジン（AI がテスト実行に使います）
npm install -g vba-runner
```

### VS Code 拡張機能のインストール（任意）

抽出した `.bas`/`.cls` ファイルをエディターで閲覧・編集する場合は、VBA Runner 拡張機能をインストールします。

VS Code の拡張機能パネル（`Ctrl+Shift+X`）で `vba-runner` を検索してインストールするか、PowerShell で実行します:

```powershell
code --install-extension jca02266.vba-runner
```

## 全体の流れ

| あなたがすること | AI がすること |
|---|---|
| 1. 抽出を指示する | `.xlsm` からテキストの VBA ソースを取り出す |
| 2. 構成を確認するよう指示する | ソースの構造・呼び出し関係を要約する |
| 3. リファクタリング/機能追加を指示する | 計画を立てて提案する（実行前に必ず確認を求める） |
| 4. 計画を承認する | テストを書きながら安全に変更する |
| 5. 書き戻しを指示する | `.xlsm` に変更を反映する |
| 6. Excel で最終確認する（あなたの作業） | — |

---

## Step 1: VBA ソースを抽出してもらう

**指示の例:**

> `MyBook.xlsm` から VBA ソースを抽出して、`src/vba` に置いてください。

AI が `.xlsm` の中身を読み取り、モジュールごとに `.bas`/`.cls` のテキストファイルとして書き出します。完了したら「どのモジュールが何個抽出されたか」を報告してもらいましょう。

---

## Step 2: ソースの構成を確認してもらう

VS Code に VBA Runner 拡張機能を入れていれば、抽出した `.bas`/`.cls` を直接開いてホバー・定義ジャンプ・コードレンズで眺めることもできますが、AI に聞いてしまうのが手早いです。

**指示の例:**

> このVBAソースの全体構成を要約してください。どのプロシージャがどこから呼ばれているか、Excel オブジェクトに直接依存している箇所はどこか教えてください。

AI は解析ツールを使って、巨大な関数や重複コード、Excel 依存箇所などを洗い出して報告します。

---

## Step 3: リファクタリングを依頼する

**指示の例:**

> このVBAソースをリファクタリングしてください。安全に進めるため、変更前に必ずテストを書いてください。

AI は次の順序で進めます。

1. 解析結果から優先順位付きの**計画**を立てる
2. 計画を提示し、**あなたの承認を待つ**（ここで承認しないと実行されません）
3. 承認後、計画の項目を1つずつ実行（テスト → 変更 → 再テスト → 効果の報告）

提案された計画が大きすぎる・対象がずれている等あれば、その場で「この項目は対象外にしてください」「もっと小さい単位に分けてください」のように伝えれば、AI が計画を修正して再提示します。

1件の計画が完了すると、AI から変更内容・テスト結果・改善効果のレポートが返ってきます。次の計画に進めるか、ここで止めるかはあなたが判断してください。

---

## Step 4: 機能追加を依頼する

リファクタリングと同じ流れで、新しい機能の追加も依頼できます。

**指示の例:**

> カテゴリ別の集計結果に「最高売上」の列を追加してください。既存の動作を壊していないことをテストで確認してから進めてください。

AI は次のように進めます。

1. 既存の動作を確認するテストがなければ、まず記録する（変更前の挙動を保証する）
2. 新機能の期待結果をテストとして先に書く（まだ実装していないので失敗する）
3. 実装する
4. テストが通ることを確認する

Excel オブジェクト（シート・セル・Application 等）に依存していてテストが難しい関数がある場合、AI はリポジトリ内のモック実装ガイドを参照して必要なモックを用意します。あなたから特別な指示は不要です。

---

## Step 5: Excel に書き戻してもらう

**指示の例:**

> 変更した内容を `MyBook.xlsm` に書き戻してください。安全のため、別名のファイルに出力してください。

AI が抽出ツールを使って `.xlsm` に書き戻します。元ファイルを直接上書きする操作には確認が必要になることがあるので、その場合は応答してください（不安なら「別名で出力してください」と毎回伝えると元ファイルは保護されます）。

---

## Step 6: Excel で最終確認する

これはあなた自身の作業です。AI は Excel を操作できません。書き戻された `.xlsm` を Excel で開き、`Alt+F11` で VBA エディターを起動して変更が反映されていることを目視確認してください。問題なければ元のファイル名に置き換えて完了です。

---

## こんな指示も使えます（応用）

- 「Excel オブジェクトに依存していて直接テストできない関数を教えてください」
- 「このリファクタリング作業のレポートをまとめてください」
- 「変更前後で行数・ネストの深さがどれだけ改善したか教えてください」
- 「次の計画に進む前に、現状のテストが全部 GREEN か確認してください」

## 困ったときは

| 状況 | 伝え方の例 |
|---|---|
| 計画の範囲が大きすぎる | 「今回は#1だけ進めてください」 |
| 既存テストが失敗した（RED） | AI が自動的に中断して報告するはずです。「なぜ失敗したか説明してください」と聞く |
| 変更後の動きに不安がある | 「Excel で確認できるよう、変更点を一覧にしてください」 |
| 元の `.xlsm` を壊したくない | 最初から「常に別名で出力してください」と伝えておく |
