# TaskScheduler — タスク自動スケジューリングマクロ

Excel 上のタスク一覧（タスク名、担当者、工数の一覧）から、**担当者ごとの日々の工数計画表**を自動生成し全タスクがいつ終わるかを計画する VBA マクロです。

> **余談**: このマクロは AI と対話しながら設計し、実装（ソースコード）は AI が作成しました。
> VBA だからか 1 つの長大な関数となりました。
> 本プロジェクトの VBA コンパイラは、この関数を AI の支援でリファクタリングするために必要に駆られて作りました（AI に作らせました）。

---

## 1. このマクロでできること

- タスク一覧のタスク実施にかかる **工数（Duration）** と **担当者** を読み取り、カレンダー列の各日に **0.25 単位（日）** で工数を割り振る
- **担当者の上限**を守りながら割当
- **休日** は自動的にスキップ
- 関連のあるタスク間では親タスクの完了予定日の後に子タスクの開始日を **自動連鎖**
- 微小な工数（0.1 など）にも特別ルールで対応
- **工数割り当てのロック** は自動割り当てを行わず手入力した割り当てのままにすることで柔軟な調整も可能

## 2. 入力（シートの準備）

実行前に、アクティブシートに以下のレイアウトを用意してください。

### 2.1 担当者キャパシティ設定（I8:J14）

| I 列（担当者名） | J 列（1 日あたりの上限） |
|----|----|
| 鈴木 | 1.0 |
| 田中 | 0.5 |
| 佐藤 |  （空欄なら 1.0 として扱う） |

> 上限を空欄にした担当者は **デフォルト 1.0** が適用されます。

### 2.2 休日行（5 行目、カレンダー列）

カレンダー（X 列以降）の **5 行目** に `休` と入れた日は割当をスキップします。
※ 土日・祝日にはあらかじめ `休` を入れておく前提です。実際には祝日カレンダーや曜日計算で Excel シート側で設定しておきます。

### 2.3 タスク行（19 行目以降）

| 列 | 名前 | 内容 |
|----|------|------|
| H | Level | `1` から始まる階層。`1` = 新ブロック / `2` 以降 = 親タスクへの依存 |
| I | Offset | 親タスクの完了からの **Lag 日数**（例: 親完了 + 2 日後に開始 → `2`） |
| M | Lock | `L` を入れるとその行は手入力を維持（自動割当しない） |
| O | Duration | 工数（例: `2.5` = 2.5 人日分） |
| Q | Assignee | 担当者名（I 列のキャパシティ設定と一致させる） |
| X 以降 | Calendar | 各日付の割当を書き込むカレンダー領域（出力先） |

## 3. 使い方

1. 上記レイアウトに従ってデータを入力（タスクは 19 行目以降に上から順に）
2. Excel の VBA エディタ（Alt+F11）で `TaskScheduler_v1.vba` をモジュールとして取り込み
3. 対象シートを **アクティブにした状態** で `AutoScheduleTasks` を実行
   - 開発タブ → マクロ → `AutoScheduleTasks` → 実行
   - またはショートカット（Alt+F8）

## 4. 得られるアウトプット

実行後、**カレンダー列（X 列以降）** に各タスクの日次工数が書き込まれます。

### 4.1 出力の見方
- 各セルに入る値は **0.25 / 0.5 / 0.75 / 1.0 ...** の倍数（通常タスク）
- 微小タスクの場合は **0.1** が 1 マスだけ入る
- 空欄 = その日には割当なし

### 4.2 出力例

タスク一覧（入力）：

| Level | Offset | Lock | Duration | Assignee |
|------|------|------|------|------|
| 1 |  |  | 1.0 | 鈴木 |
| 2 |  |  | 0.5 | 鈴木 |
| 2 | 1 |  | 0.5 | 田中 |

カレンダー（出力、田中の上限は 0.5）：

| Task | 1 日目 | 2 日目 | 3 日目 |
|------|------|------|------|
| 1 (鈴木, 1.0) | 1.0 |  |  |
| 2 (鈴木, 0.5) |  | 0.5 |  |
| 3 (田中, 0.5 + Lag1) |  |  | 0.5 |

- タスク 1 は鈴木が 1 日目で消化（上限 1.0）
- タスク 2 は親（タスク 1）の完了割当 `1.0 ≧ 0.5` のため **翌日開始**
- タスク 3 は親完了の翌日が基本開始だが、Offset = 1 で **さらに 1 日後** から開始

## 5. 主なルール

| ルール | 内容 |
|------|------|
| 同日 / 翌日開始の判定 | 親タスクの最終日の割当が **0.5 未満** なら同日開始、それ以外は翌日開始 |
| 割当単位 | 通常 **0.25**、微小タスクは **0.1**（1 日のみ） |
| キャパシティ既定値 | I/J 列に未指定の担当者は **1.0** |
| 休日 | カレンダー 5 行目に `休` のある日はスキップ |
| ロック行 (`M = L`) | カレンダー値を保持しつつ使用量に集計、後続タスクの空き容量計算に反映 |
| Level 1 | これまでの依存を **リセット**（新しいタスクブロックの開始） |

## 6. 動作上の注意

- 実行中は **画面更新・自動計算・イベント**を一時停止します（処理後に自動復元）
- カレンダー列の **既存値は上書き** されます（ロック行を除く）
- 担当者の上限を超える工数のタスクは複数日に分割して割り当てられます
- タスクの工数が大きすぎてカレンダー範囲内に収まらない場合、入りきらなかった分はスケジュールされません
